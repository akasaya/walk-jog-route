# 要件定義: route-suggestion-mvp

**フィーチャー名**: route-suggestion-mvp  
**説明**: ルート提案 MVP  
**ステータス**: 要件生成済み（承認待ち）  
**作成日**: 2026-05-19

---

## 概要

現在地から散歩・ジョギングコースを AI が自動提案する Web アプリの MVP。
ユーザーの過去ルート履歴・現在の天気・現在地を文脈として AI がルートを提案し、
地図上にポリラインで表示する。

---

## 1. 現在地取得

**1.1** ウォークジョグルートアプリは、ユーザーが提案ページを開いたとき、ブラウザの位置情報 API を通じて現在地（緯度・経度）を取得すること。

**1.2** ユーザーが位置情報の許可を拒否したとき、ウォークジョグルートアプリは位置情報が必要である旨を通知し、手動入力の代替手段を提供すること。

**1.3** ウォークジョグルートアプリは、位置情報の取得に失敗したとき（タイムアウト・エラー）、エラーメッセージを表示してユーザーが再試行できること。

---

## 2. ルート提案

**2.1** ユーザーが移動モード（散歩 / ジョギング）と目標距離（km 単位）を指定して提案を依頼したとき、ウォークジョグルートアプリは 30 秒以内にルート候補を返すこと。

**2.2** ルート提案は、以下のすべてを入力として考慮すること：
- ユーザーの現在地
- 目標距離・移動モード
- 直近の過去ルート履歴（直近 10 件以内）
- 提案時点の天気・気温

**2.3** ウォークジョグルートアプリが提案するルートは、ユーザーの現在地をスタート兼ゴールとする周回コースであること。

**2.4** ウォークジョグルートアプリが提案するルートの総距離は、ユーザーが指定した目標距離の ±20% 以内であること。

**2.5** ルート提案システムは、ユーザーの過去履歴を参照し、過去に通行したルートと異なる経路や未訪問エリアを優先して提案すること。

**2.6** ルート提案に 30 秒以上かかる場合、または外部サービスがエラーを返した場合、ウォークジョグルートアプリはエラーメッセージを表示し、ユーザーが再試行できること。

**2.7** ウォークジョグルートアプリは、目標距離をスライダーまたは数値入力フィールドで自由に指定できること。入力範囲は 0.5 km 以上 50 km 以下とし、範囲外の値は入力時にバリデーションエラーを表示すること。

---

## 3. 地図表示

**3.1** ウォークジョグルートアプリは、提案されたルートを地図上にポリラインとして表示すること。

**3.2** ウォークジョグルートアプリは、現在地マーカーを地図上に表示すること。

**3.3** ウォークジョグルートアプリは、地図の初期表示範囲をルート全体が収まるように自動調整すること。

**3.4** 地図はインターネット接続がある環境で利用可能であること。オフライン時は「地図を表示できません」と表示すること。

---

## 4. ルート実行・GPS 記録

**4.1** ユーザーが提案されたルートの「開始」を操作したとき、ウォークジョグルートアプリはそのルートをデータストアに保存すること。

**4.2** ルート実行中、ウォークジョグルートアプリはブラウザタブが開いている間、定期的にユーザーの現在位置を記録すること。

**4.3** ウォークジョグルートアプリは、ユーザーがルートを「完了」または「中断」と操作したとき、記録を終了してデータを保存すること。

**4.4** ルート実行中、ウォークジョグルートアプリは地図上にユーザーの現在位置を継続的に表示すること。

---

## 5. ルート履歴

**5.1** ウォークジョグルートアプリは、過去に開始したルートの一覧を表示すること。

**5.2** 各ルート履歴には、少なくとも以下を表示すること：
- 実施日時
- 移動モード（散歩 / ジョギング）
- 計画距離（km）
- 実績記録の有無

**5.3** ユーザーが履歴の特定ルートを選択したとき、ウォークジョグルートアプリはそのルートを地図上に表示すること。

---

## 6. 天気情報の取得

**6.1** ウォークジョグルートアプリは、ルート提案の実行時に現在地の天気情報（気温・天気状況）を自動取得すること。

**6.2** 天気情報の取得に失敗したとき、ウォークジョグルートアプリは天気情報なしでルート提案を継続すること（天気情報はオプションとして扱う）。

---

## 7. 非機能要件

**7.1** ウォークジョグルートアプリは、最新版の Chrome / Firefox / Safari に対応する Web アプリとして動作すること。

**7.2** ルート提案の API は、入力バリデーションに失敗した場合 HTTP 422 を返すこと。緯度は -90〜90、経度は -180〜180、距離は 0.5〜100 km の範囲を有効とすること。

**7.3** ルート提案の API は、外部サービス（天気・ルーティング・AI）が利用不能な場合 HTTP 503 を返すこと。

**7.4** ウォークジョグルートアプリは、ユーザーの位置情報（GPS 座標）を詳細な精度でログ・エラーメッセージに含めないこと。

**7.5** ルート提案 API は、同一クライアントからの連続リクエストに対してレートリミットを適用すること。
