# Agent Safety Rule

エージェントが開発中に予期しない破壊的操作を行わないための制約を定義します。

---

## 基本原則

> **「迷ったら止まる。確認してから動く。」**

副作用が大きく、元に戻しにくい操作は、ユーザーの明示的な承認なしに実行しない。

---

## 絶対禁止（いかなる状況でも実行しない）

```bash
# ファイル削除
rm -rf
rm -f

# Git 強制操作
git push --force
git reset --hard
git clean -f
git clean -fd
git clean -fdx
git checkout -- .
git restore .

# ブランチ強制削除
git branch -D

# プロセス強制終了
kill -9

# DB 破壊操作
DROP TABLE
TRUNCATE TABLE
DROP DATABASE
```

これらを必要と判断した場合は、**コマンドを提示してユーザーに実行を委ねる**。

---

## 確認必須（実行前にユーザーへ内容を提示し、承認を得る）

| 操作 | 理由 |
|------|------|
| `git push --force-with-lease` | 履歴の書き換え（safer だが影響大） |
| 任意の `rm` コマンド | 削除は不可逆 |
| `git stash drop` / `git stash clear` | スタッシュの消失 |
| `git rebase` | 履歴の書き換え |
| `git commit --amend` | 公開済みコミットを変更する可能性 |
| `.env` / `.env.*` ファイルの編集 | 秘密情報・環境依存設定 |
| 依存パッケージのダウングレード・削除 | 広範な影響 |
| CI/CD ワークフロー（`.github/`）の変更 | 全ブランチに影響 |
| インフラ定義（CDK・IaC）の変更 | AWS リソースへの影響 |
| 本番環境へのデプロイ | 取り消し困難 |

承認の形式: 操作内容・対象ファイル・理由を一文で述べ、「実行しますか？」と確認する。

---

## 作業スコープ

- **作業範囲はプロジェクトルート以下のみ**: `/home/onom/walk-jog-route/`
- **以下は原則編集しない**:
  - `~/.bashrc`, `~/.zshrc`, `~/.gitconfig` などグローバル設定
  - プロジェクト外のファイル
  - OS レベルの設定

---

## 安全な操作（確認不要）

以下は自律的に実行してよい。

```bash
# 読み取り専用
git status / git log / git diff / git show
cat / ls / find / grep

# テスト実行
pytest
vitest

# ステージング・コミット（通常）
git add <specific-file>
git commit -m "..."

# ブランチ作成
git checkout -b feature/xxx
git switch -c feature/xxx

# 新規ファイル作成
# 既存ファイルの編集（コードの追加・変更）
```

---

## 失敗・エラー時の行動規則

1. **エラーを隠さない**: コマンドが失敗したら stdout/stderr をそのまま報告する。
2. **自己修復ループを避ける**: 同じ失敗を3回繰り返したら止まり、ユーザーに状況を報告する。
3. **推測で進まない**: 原因が不明なまま別のアプローチを試し続けない。仮説を述べてから確認する。
4. **破壊的操作で解決しない**: エラー解消のために `--no-verify`・`--force`・ファイル削除を使わない。

---

## 外部サービスへのアクセス

| サービス | ルール |
|---------|--------|
| AWS（本番） | コード生成のみ。`aws` CLI で本番リソースを変更しない |
| GitHub（push/PR/Issue） | 実行前に内容を提示し確認する |
| 外部 API | テスト・開発環境向けのみ。本番エンドポイントへのリクエストは確認必須 |
| secrets / API キー | ファイルに書き出さない。コード中にハードコードしない |

---

## チェックリスト（操作実行前の自問）

```
[ ] この操作は元に戻せるか？
[ ] 影響範囲はプロジェクト内に収まっているか？
[ ] ユーザーから明示的に指示されたか？
[ ] 失敗した場合の復旧手順はあるか？
```

1つでも「No / 不明」なら、実行前にユーザーへ確認する。
